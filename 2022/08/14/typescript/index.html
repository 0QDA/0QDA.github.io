TS变量声明

  • 类型声明

    • 类型声明是TS非常重要的一个特点

    • 通过类型声明可以指定TS中变量（参数、形参）的类型

    • 指定类型后，当为变量赋值时，TS编译器会自动检查值是否符合类型声明，符合则赋值，否则报错

      let 变量: 类型;
let 变量: 类型 = 值;
let a: number
a = 10

  • 自动类型判断

    • TS拥有自动的类型判断机制
    • 当对变量的声明和赋值是同时进行的，TS编译器会自动判断变量的类型
    let a=10
let b="qqdl"

声明示例

  1. 声明一个变量a指定类型为number/string/boolean

    let a: number
let b=10
a = 10

let b: string
b = 'qqdl'

let c: boolean = false
c = true

  2. 声明方法，指定传入参数类型以及返回类型

    function sum(x: number, y: number): number {;
 return x + y
};
console.log(typeof sum(1, 2))

  3. 可以使用 | 来连接多个类型（联合类型）

    let d: 'male' | 'female'
d = 'male'

  4. 字面量

e = true
e = 'qqdl'
let color: 'red' | 'blue' | 'black'

    let f: any
let g

  4. 声明对象类型：{}用来指定对象中可以包含那些属性，语法：{属性名：属性值，属性名：属性值} 在后面加上？表示属性是可选的

面向对象

类(class)

定义类

class 类名 {;
    属性名: 类型;
    constructor(参数: 类型){;
        this.属性名 = 参数;
    };
    方法名(){;
        ....
    };
};

示例

class Dog {;
  name: string
  age: number
  constructor(name: string, age: number) {;
    // 构造函数中this就是当前创建的对象
    this.name = name
    this.age = age
  };
  talk() {;
    // this 表示的当前方法
    console.log(this)
    console.log('你在狗叫什么')
  };
};

使用

const dog1 = new Dog('白', 3)
const dog2 = new Dog('黑', 4)
dog1.talk();
console.log(dog1)
console.log(dog2)

面向对象的特点

继承

class Amimal {;
  name: string
  age: number
  constructor(name: string, age: number) {;
    this.name = name
    this.age = age
  };
  say() {;
    console.log('你在叫什么')
  };
};
// 继承
class Dog extends Amimal {;
  say(): void {;
    console.log('你在狗叫什么')
  };
};
class Cat extends Amimal {;
  master: string
  constructor(name: string, age: number, master: string) {;
    super(name, age) //调用父类的构造函数
    this.master = master
  };
  say() {;
    console.log(`${;this.master};主人加油哦`)
  };
  cook() {;
    console.log('你他喵有病是吧')
  };
};
const dog = new Dog('黑', 2)
const cat = new Cat('白', 3, '秋去冬来')
dog.say()
cat.say()

封装

对象实质上就是属性和方法的容器，它的主要作用就是存储属性和方法，这就是所谓的封装

  • 默认情况下，对象的属性是可以任意的修改的，为了确保数据的安全性，在TS中可以对属性的权限进行设置(public)

  • 只读属性（readonly）：

    • 如果在声明属性时添加一个readonly，则属性便成了只读属性无法修改

  • TS中属性具有三种修饰符：

    • public（默认值），可以在类、子类和对象中修改
    • protected ，可以在类、子类中修改
    • private ，可以在类中修改
class Person {;
  // public默认，表示任意位置可以访问(修改)
  // private私有属性只能在类中进行修改
  // protected受保护属性，只能在当前类和子类中访问和修改
  private _name: string
  constructor(name: string, age: number) {;
    this._name = name
  };
  //默认设置get set方法
  // getName() {;
  //   return this.name
  // };
  // setName(value: string) {;
  //   this.name = value
  // };
  // ts设置设置Getter setter方法
  get name() {;
    return this._name
  };
  set name(value: string) {;
    this._name = value
  };
};
class Amianl {;
  constructor(public name: string) {;};
};
const per = new Person('秋去冬来')

接口(Interface)

接口的作用类似于抽象类，不同点在于接口中的所有方法和属性都是没有实值的，换句话说接口中的所有方法都是抽象方法。接口主要负责定义一个类的结构，接口可以去限制一个对象的接口，对象只有包含接口中定义的所有属性和方法时才能匹配接口。同时，可以让一个类去实现接口，实现接口时类中要保护接口中的所有属性

// 描述一个对象的类型
type myType = {;
  name: String
  age: number
};
// 接口用来定义一个类结构 用来定义一个类应该包含那些属性和方法，可以当成类型声明去使用
// 接口中的所有属性都不能有实际值 只定义对象结构，不考虑实际值
interface myInterface {;
  name: String
  say(): void
};
//可以重复的名字
interface myInterface {;
  age: number
};
// 实现接口
class MyCalass implements myInterface {;
  name: string
  age: number
  constructor(name: string, age: number) {;
    this.name = name
    this.age = age
  };
  say() {;
    console.log('向前所未见的明天疾驰')
  };
};

泛型(Generic)

定义一个函数或类时，有些情况下无法确定其中要使用的具体类型（返回值、参数、属性的类型不能确定），此时泛型便能够发挥作用。

// 定义函数或者类时，遇到类型不明确时可以使用泛型
// 可以直接调具有泛型的函数
function fn<T>(a: T): T {;
  return a
};

// 泛型可以同时指定多个
function fn2<T, K>(a: T, b: K): T {;
  console.log(b)
  return a
};
let res = fn(10) //不指定泛型，TS可以自动推断
let res2 = fn<string>('秋去冬来') //指定泛型
console.log(res, res2)

fn2<number, string>(123, 'qqdl')

interface Inter {;
  length: number
};
// 表示泛型T必须是Interd 实现类(子类)
function fn3<T extends Inter>(a: T): number {;
  return a.length
};
fn3('123')

class MyCalss<T> {;
  constructor(public name: T) {;};
};

const mc = new MyCalss<string>('秋去冬来')
